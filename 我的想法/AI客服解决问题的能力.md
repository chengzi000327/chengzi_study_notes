# AI客服解决问题的能力：评测与兜底机制

2026-05-28

## 核心问题

AI客服的核心不是"多聪明"，而是**有没有解决问题的能力**——能帮助平台解决用户和商户的问题。

## 关键设计要素

1. **兜底机制**：给AI客服设置明确的权限边界，超出边界要有降级路径
2. **能力评测**：如何衡量"解决了"？

## "解决了"的三方定义

| 角色 | "解决了"的信号 |
|---|---|
| 用户 | 拿到了补偿/方案 + 情绪平复 + 不再找客服 |
| 商家 | 用户不再追责 + 成本可控 |
| 平台 | 双方都没流失 + 成本 < 人工 + 留痕 |

## 兜底三层设计

- **第一层（80%）**：AI自主决策——查单、改地址、标准退款、FAQ
- **第二层（15%）**：AI + 规则引擎——预设条件组合决策（如VIP+首申→自动授权50元内补偿）
- **第三层（5%）**：AI + 人工——AI做context摘要+推荐方案，人工一键确认

> 小橙子注：这个框架的关键洞察在于——"解决问题"在用户和商家侧往往定义不同甚至对立，评测体系必须同时追踪双方的信号。
