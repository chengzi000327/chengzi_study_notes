# 独立思考与决策框架的三层递进

> 从正职的反馈提炼——三条讲的是同一件事的递进：从"我该听谁的"到"我的判断是什么"

## 三条的拆解

### 1. 独立思考 + 有原则地做决策

不是拍板定方案，而是：

- **有判断**：别人给意见时能分辨"合不合理"，而不是谁说什么就改什么
- **有论据**：说"我认为应该这样做"时能说出"因为xxx"（用户行为数据/竞品逻辑/业务目标）
- **敢拒绝**：需求/建议明显不合理时，能说"这样做对xx没帮助，因为用户没有参与成本"

> 本质：有自己的立场再去对齐

### 2. 做长期正确的事 + 阶段性平衡

- **长期正确**：未来要做完整的产品功能 map
- **阶段性平衡**：现在只做其中一个点，因为其他部分还没那么重
- **规划思维**：做某个功能时，设计要考虑"后面哪些功能会需要"，现在就做通用结构
- **操作翻译**：每做一个功能问自己——"这东西做完，下一期能不能直接复用？如果不能，多花10%成本让它能复用？"

**隐藏前提**：你得先知道"下一期要做什么"。没有自己的产品 map，你无法判断那"10%该花在哪里"。所以第2条和第1条联动——**没有独立的判断力，连"怎么复用"都看不准**，只会做成"伪通用"的重架构。

### 3. 实习生身份的转化

**陷阱循环**：
收到不同意见 → "我是实习生，可能人家说得对" → 全盘接受 → 改来改去 → 方案失去主线

**期望模式**：
收到不同意见 → 判断"这个意见的逻辑是什么？和我的方案冲突在哪？" → 如果对方有道理，吸收；没道理，用论据驳回 → 方案有主线

**根因拆解**：
不是态度问题，而是没有建立决策坐标。实习生的"学习"不是学怎么执行，是学怎么判断。方案改来改去不是因为能力不行，是因为还没建立"什么情况下该坚持、什么情况下该妥协"的判断标准。

## 我的补充操作维度

每次收到意见时问自己：
- 这个意见是**事实层面**的（用户行为数据有出入）还是**偏好层面**的（我觉得这样更好）？
- 事实层→直接改；偏好层→启动"判断+论据+拒绝"流程

比"用全盘接受压住焦虑、最后再改回来"效率高得多。

## 三层闭环

独立思考 → 有产品map → 阶段性平衡 → 复用设计
