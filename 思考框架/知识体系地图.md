# 🗺️ 橙子的知识体系地图

> 跨论文原理网络。每一层 = 一个底层原理，下一级 = 二级概念/具体实现。
> 新知识进来时，定位到对应原理节点下。

---

## 一、基础层：AI Agent 的核心约束

### 原理1：上下文窗口是硬约束
背后逻辑：LLM 能同时看到的信息量有限，所有设计都必须在这个约束下做优化。

| 二级概念 | 论文/项目 | 关键洞察 |
|---|---|---|
| 上下文即内存 | MemGPT | 像 OS 管理虚拟内存一样管理上下文，swap 换入换出 |
| 渐进式披露 | Agent Skills 综述 | 元数据预加载，触发时才加载完整正文 |
| 约束解码 | ToolGen | 只允许生成真实存在的工具 token，幻觉归零 |

**↔ 关联：** 上下文管理（C层）← 直接影响工具选择（T层）

### 原理2：状态管理决定Agent一致性
背后逻辑：Agent 能否记住并保持一致，取决于状态的存储、更新和遗忘机制。

| 二级概念 | 论文/项目 | 关键洞察 |
|---|---|---|
| 记忆分层 | Generative Agents | 短期→长期→抽象，三层记忆架构 |
| 遗忘机制 | MemoryBank | 艾宾浩斯遗忘曲线，时间衰减 |
| ADD/UPDATE/DELETE | Mem0 | 不是记住一切，是记得住该记得的 |
| 文件系统即状态 | Claude Code | 把状态存在文件系统里而非内存里 |

**↔ 关联：** 状态管理（C层）← 决定编排（L层）能否跨会话协作

---

## 二、模型层：Agent 的能力边界

### 原理3：工具模型比语言模型更关键
背后逻辑：决定 Agent 行为上限的不是模型本身，而是模型外面的包装（Harness）。

| 二级概念 | 论文/项目 | 关键洞察 |
|---|---|---|
| Binding-constraint 论点 | Agent Harness Engineering | 改工具格式比换模型效果更显著（10x提升） |
| 检索即能力 | Gorilla | 离线训练追不上线上变化，推理时检索更可靠 |
| 检索生成一体化 | ToolGen | 拆两步不如合成一步，消除误差累积 |

**↔ 关联：** Harness（ETCLOVG全部七层）← 约束条件决定了模型能发挥多少能力

### 原理4：自我学习优于人工标注
背后逻辑：让模型自己学会何时调用工具、何时更新记忆，替代人工规则。

| 二级概念 | 论文/项目 | 关键洞察 |
|---|---|---|
| 困惑度筛选 | Toolformer | 工具调用必须"真的帮到了语言建模"才算有效 |
| 数据说话 | Toolformer | 不问用户要什么，看行为上的实际受益 |
| 记忆命中率 | Mem0 | 召回是否符合用户预期，数据驱动优化 |

**↔ 关联：** 自我学习（T层+Skills层）← 依赖评估（V层）来验证效果

---

## 三、技术层：多Agent系统的工程实践

### 原理5：角色分工是协作的基石
背后逻辑：多Agent不是堆更多模型，而是定义清楚谁做什么。

| 二级概念 | 论文/项目 | 关键洞察 |
|---|---|---|
| 角色提示 | CAMEL | "Never flip roles!" 维持角色边界的提示设计 |
| SOP驱动 | MetaGPT | 工作流规范化，每个Agents的输入/输出格式事先定义 |
| 容错循环 | AutoGen | 能发现错误并自我修复，比一次正确更重要 |

**↔ 关联：** 角色（L层）← 每个角色的工具权限由（G层）定义

### 原理6：结构化输出减少级联错误
背后逻辑：Agent之间传递标准格式的交付物，而非自由文本，防止级联幻觉。

| 二级概念 | 论文/项目 | 关键洞察 |
|---|---|---|
| 标准化中间产物 | MetaGPT | PRD→设计→任务→代码→测试，每步有验证点 |
| 共享消息池 | MetaGPT | 减少无效通信，Agent订阅自己关心的消息 |
| 编排优于自治 | 面试题分析 | 好PM设计的是让不聪明Agent正确协作的框架 |

**↔ 关联：** 标准化（T层+L层）← 可观测性（O层）帮助验证中间产物质量

## 四、应用层：产品视角

### 原理7：信任来自一致性
背后逻辑：用户信任AI不是因为它聪明，而是因为它稳定可靠。

| 二级概念 | 论文/项目 | 关键洞察 |
|---|---|---|
| 记忆一致性 | Generative Agents | 昨天答应的事今天还记得 |
| 品牌一致性 | AI助手功能边界题 | 闲聊/情感陪伴不能破坏工具产品的品牌定位 |
| 容错体验 | 多Agent面试题 | 降级路径也须输出用户可见的完整回复 |

### 原理8：安全是设计内嵌，不是事后补丁
背后逻辑：Agent Skills 26.1% 存在安全漏洞说明安全审核是平台级责任。

| 二级概念 | 论文/项目 | 关键洞察 |
|---|---|---|
| 分级授权 | Agent Skills 综述 | 按来源可信度授予权限 |
| 五层防护 | AI助手功能边界题 | 实时安全管道→危机干预→人格护栏→用户控制→风险仪表盘 |
| 三阶段灰度 | AI助手功能边界题 | 冷启动1%→扩量5-20%→全量，每阶段有退出阈值 |

---

## 跨层核心循环

```
基础层约束（上下文/状态）→ 模型层边界（工具/自学习）
→ 技术层实践（角色/结构化）→ 应用层产品（信任/安全）
→ 反馈回到基础层（可观测性数据驱动改进）
```

**这个循环每走一圈，你的知识体系就更像一个专家的原理网络。**
