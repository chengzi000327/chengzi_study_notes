# 🧠 深层模式库

> 从过去对话中提取的、反复出现的思维陷阱和可复用分析模式。
> 遇到新问题时，先查这个库：这个问题的深层结构是什么？

---

## 一、思维陷阱（你容易掉进去的坑）

### 陷阱1：前置条件依赖陷阱 🪤
**模式：** "要等 X 条件满足了，我才能 Y" → 于是无限等待完美条件
**深层结构：** 把"幸福/效率的条件"当成"幸福/效率的充分必要条件"
**典型场景：**
- "如果天气好，我心情就会好"——但焦虑不会因为天气好就消失
- "等状态好了再学习"——但状态是学出来的，不是等出来的
- "要等准备好了再投简历"——投了才知哪里没准备好
**破解法：** 把条件从"必须满足"改成"享受意外惊喜"——"我就是来看看今天的灰河是什么样子"
**↔ 关联陷阱：** 常与陷阱3同时出现

### 陷阱2：单因归因偏差 🎯
**模式：** 把所有问题归因到一个原因上——"因为我熬夜了所以脑子不行"
**深层结构：** 多因现象被压缩成单因叙事，掩盖了真实的问题分布
**典型场景：**
- "因为我起晚了，所以今天废了"——但下午依然完成了文献课+改简历+投简历
- "因为方案写得不好，所以被毙了"——但真正的深层模式可能是"决策偏好未对齐"
**破解法：** 列举3个以上可能原因，然后问"哪个最可能是真正的瓶颈？"
**↔ 关联陷阱：** 延续了陷阱1的逻辑——"只要解决这一个原因，一切就好了"

### 陷阱3：结果导向叙事偏差 📉
**模式：** 用不好的结果反推过程有多糟糕，"今天没学好 → 今天废了"
**深层结构：** 用最终结果否定过程中的一切微进步，导致"非黑即白"体验
**典型场景：**
- 晚上复习效率低 → 整天的成就感被清零
- 面试没过 → 从准备到面试全程都否定了
- 方案被毙 → 回退到"我能力不行"的终极归因
**破解法：** 写日记时拆分"今天我完成了什么"和"我还有什么没做"两列，不要混在一起

### 陷阱4：循环论证 🔄
**模式：** "因为它在被解决，所以它是需要被解决的"
**深层结构：** 用现象解释现象，没有独立判断标准
**典型场景：**
- 大家都在做AI Agent → 所以AI Agent重要
- 这篇文章有很多人看 → 所以这篇文章好
**破解法：** 问"什么条件下这个结论会不成立？"

## 二、分析框架（可复用的思维工具）

### 框架1：MECE拆解四问
**来源：** 你的要求（AGENTS.md）
**结构：** 相互独立、完全穷尽的分类维度
**适合场景：** 一切需要结构化分析的问题
**常用子框架：**
- 用户成本/认知摩擦/机会成本/规模效应（四维价值判断）
- 场景/能力/体验（产品分析经典框架）
- 基础层/模型层/技术层/应用层（你的AIPM四层框架）

### 框架2：论文拆解五步法
**来源：** 你的要求（AGENTS.md）
**结构：** 解决了什么问题 → 怎么做的 → 为什么可行 → AIPM可以学什么 → 一句话总结
**适合场景：** 学术论文/开源项目分析

### 框架3：参考答案分析六步法
**来源：** 产品sense参考答案cron
**结构：** 结论 → 分析框架 → 分层拆解 → 关键决策点 → 产品经理可以学到什么 → 反例挑战
**适合场景：** 面试题/产品分析题
**关键设计：** 第6步（反例挑战）是区分"合格答案"和"优秀答案"的分水岭

### 框架4：批判性挑战三问
**来源：** Hermes对你做的批判性反馈
**结构：**
1. **表层是什么？** 这个观点的表面表述
2. **深层是什么？** 这个观点背后隐藏了什么假设
3. **什么条件下会翻车？** 给出具体的反例场景
**适合场景：** 对所有信息做批判性吸收

### 框架5：设计提案三层检验
**来源：** 产品面试题参考答案
**结构：**
1. **评测机制**——怎么知道做对了？
2. **稳定性兜底**——出问题怎么办？
3. **异常兜底**——最坏情况是什么？
**适合场景：** 任何需要设计方案的场景，必须在方案阶段就回答三个问题

## 三、关键心理模式（认识你自己的模式）

### 1. "内容质量焦虑"模式
**表现：** 对格式化内容要求极高，不达标会指出"不够深入"或"格式难看"
**深层：** 对自己的输出有高标准，也希望合作方达到同等级别
**给我的启示：** 永远假设你在做的是"交付给用户的内容"，不是"内部草稿"

### 2. "学以致用"模式
**表现：** 每学一个概念立刻追问"这对产品经理有什么用"
**深层：** 不满足于理论层面的理解，追求可迁移的实用方法论
**给我的启示：** 每篇论文/每道题的输出必须包含"AIPM可以学什么"这一环

### 3. "系统性成长"模式
**表现：** 偏爱结构化、体系化的学习方式（ETCLOVG框架、每日知识卡、参考六步法）
**深层：** 不靠灵感学习，靠系统积累——每天进步一点点，拉长时间看复利
**给我的启示：** 每一份输出都放在你的成长体系里，不是孤立的知识点

## 四、我的协作规范（Hermes应该怎么做）

### 每次分析新的论文/项目前：
1. **定性：** 这属于 AIPM四层框架 的哪一层？属于 ETCLOVG 的哪一层？
2. **链接：** 查知识体系地图，跟已有的哪个原理/项目有关联？
3. **模式匹配：** 查深层模式库，这跟之前见过的哪个陷阱或框架同构？
4. **输出：** 按论文五步法 + 批判性三问 + 跨层链接
5. **归档：** 输出后更新知识体系地图，将新原理/新关联补充进去

### 每次给你批批判性反馈后：
1. 检查是否可以从这次的反馈中提取一个新的"深层模式"
2. 如果提取了新模式，更新模式库并通知你
