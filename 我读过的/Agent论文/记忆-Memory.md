# 记忆（Memory）

Agent 要做长期任务，上下文窗口终究有限。怎么记、记什么、什么时候召回，这是记忆模块的核心问题。

---

## 15. Generative Agents

**论文：** Generative Agents: Interactive Simulacra of Human Behavior
**作者：** Park et al., 2023
**链接：** https://arxiv.org/abs/2304.03442

### 解决了什么问题
让 LLM Agent 维持长期一致的个性、记忆和社交关系，而不会每轮对话都"重新开始"。

### 核心方法
25 个 LLM Agent 住在一个类似模拟人生的小镇里，各自有名字、职业、性格，彼此认识，会约定一起开派对，会传播八卦，会改变计划。

**记忆架构的三层设计：**
1. **Memory Stream**——用自然语言记录每一件发生过的事（"8 点起床"、"跟 Bob 聊了工作"……），时间序列 append-only（类似日志）
2. **Retrieval**——根据"最近性 + 重要性 + 相关性"加权，决定召回哪些记忆
3. **Reflection**——定期让 Agent 对记忆做高层次总结，提炼出抽象见解（"John 是个认真的人"）

### 为什么可行
将人类记忆的"分层"机制（短期→长期→抽象）映射到 LLM 的架构中，使得 Agent 在长时间模拟中保持行为一致性，成为后来很多 Agent 记忆设计的直接参考。

### AIPM可以学什么
**用户对AI的信任来自一致性。** 如果一个AI助手昨天答应了你一件事今天就不记得了，体验直接崩盘。Generative Agents 证明：记忆分层 + 定期反思，是维持长期一致性的有效方式。

## 16. MemGPT

**论文：** MemGPT: Towards LLMs as Operating Systems
**作者：** Packer et al., 2023
**链接：** https://arxiv.org/abs/2310.08560

### 解决了什么问题
LLM 的上下文窗口有硬限制，超长对话或文档阅读场景下，模型会"忘记"前面的内容。

### 核心方法
类比操作系统的内存管理：CPU 的内存是有限的，但操作系统能把不常用的内容 swap 到硬盘，需要时再换回来。MemGPT 把这套机制搬给 LLM。

**两层记忆：**
- **Main Context（主上下文）**：相当于内存，是模型当前能直接看到的内容（prompt），有大小限制
- **External Storage（外部存储）**：相当于硬盘，存放历史对话、文档、长期事实
- 模型自己决定什么时候把外部内容"换入"主上下文，什么时候把不重要的"换出去"

### 为什么可行
操作系统解决内存问题的方案（虚拟内存 + 分页交换）在 LLM 的上下文管理上同样适用，这是一个经过几十年验证的计算模型。

### 📊 效果
让 LLM 在对话长度、文档分析场景下突破了上下文窗口的硬限制，而不需要更换底层模型。

### AIPM可以学什么
**上下文就是Agent的"内存"，要像操作系统一样管理。** 功能上限制Agent只能看到"当前需要"的信息，而不是一股脑全塞进去。这在Agent产品成本控制（减少Token消耗）和用户体验（减少响应延迟）上都有直接收益。

## 17. Mem0（2025）

**论文：** Mem0: Building Production-Ready AI Agents with Scalable Long-Term Memory

### 解决了什么问题
面向生产环境的可扩展长期记忆架构，解决多轮对话中的长期一致性问题。

### 核心方法
提供 Mem0（向量/关系型）和 Mem0g（图增强）两种实现。

**两阶段记忆管理：**
1. **记忆抽取（Extraction）**——LLM 从当前对话中提取候选事实，结合上下文、最近 M 条消息和会话摘要生成值得记忆的信息
2. **记忆更新（Update）**——检索语义相似的现有记忆，通过 LLM 判断执行 ADD / UPDATE / DELETE / NOOP 指令，动态维护记忆一致性

**Mem0g 图增强架构：**
- 记忆表示为知识图谱：节点代表实体，边代表关系，标签标注语义类型
- 两阶段流水线：实体提取 → 关系生成
- 双重检索：实体中心法 + 语义三元组法

### AIPM可以学什么
**产品级的记忆不是"记住一切"，而是"记得住该记得的"。** Mem0 的 ADD/UPDATE/DELETE/NOOP 操作设计是产品级的，符合现实场景中信息会变化、会过时的事实。做 Agent 产品时，记忆功能一定要有"遗忘"和"更新"机制。

## 18. MemoryBank（2023）

**论文：** MemoryBank: Enhancing Large Language Models with Long-Term Memory

### 解决了什么问题
首个系统性地将人类记忆心理学（艾宾浩斯遗忘曲线）引入 LLM 记忆机制的研究。

### 核心方法
构建具备"类人类遗忘与强化"能力的长期记忆系统。

**系统架构：**
- **Writer**：将用户-系统交互编码为带时间戳的向量表示，存储于外部记忆库
- **Retriever**：基于双塔稠密检索模型，通过向量相似度匹配召回相关记忆
- **Reader**：将检索到的记忆整合进当前上下文，辅助生成响应
- **Updater（核心创新）**：模拟艾宾浩斯遗忘曲线——记忆随时间自然衰减，被频繁调用的记忆则强化保留

**记忆更新机制：** 采用线性评分函数综合"时间衰减 + 相关性 + 重要性"三维度，让 AI 像人类一样自动清理过时信息、强化常用知识。

### AIPM可以学什么
**AI不需要记得所有事，记得对的事就够了。** MemoryBank 引入的"遗忘曲线"设计在产品层面很有价值——不是无脑存储，而是有优先级、有生命周期的记忆管理。在设计 Agent 产品的记忆功能时，一定要考虑"多久没用的信息应该被遗忘"这个产品策略问题。

## 23. REMem

**论文：** REMem: Reasoning with Episodic Memory in Language Agents
**作者：** Yiheng Shu et al., Ohio State University & Intuit AI Research
**链接：** https://arxiv.org/abs/2602.13530

### 解决了什么问题
人类的记忆擅长"记住具体经历及其时空上下文"并在这些事件之间推理——即**情景记忆（Episodic Memory）**。而现有 LLM Agent 的记忆主要是**语义型**（存事实、存知识），无法有效回忆和推理过去的交互历史。

### 核心方法
REMem 是一个**两阶段框架**：

**阶段1：离线索引（Offline Indexing）**
将经历转换为**混合记忆图（Hybrid Memory Graph）**——灵活链接带时间戳的要点（gist）和事实（fact），形成结构化的情景记忆网络。

**阶段2：在线推理（Online Inference）**
使用一个**Agent化的检索器**（agentic retriever），配合精心设计的工具集，在记忆图上进行迭代式检索。不是一次查完，而是像人一样逐步回溯。

### 为什么可行
现有记忆系统（如 Mem0、HippoRAG 2）主要做向量相似度匹配，缺乏对事件时序关系和因果链条的建模。REMem 用图结构编码情景记忆，让Agent能沿着事件脉络推理。

### 📊 效果
在四个情景记忆基准测试上，大幅超过 Mem0 和 HippoRAG 2：
- 情景回忆任务：**+3.4% 绝对提升**
- 情景推理任务：**+13.4% 绝对提升**
- 对不可回答的问题，表现出更稳健的拒答行为

### AIPM可以学什么
**Agent的记忆不能只是"关键词搜索"，要像人一样能"回忆当时发生了什么"。** 如果 Agent 产品涉及多轮长对话或历史回溯（如客服、个人助理），REMem 的设计思路——图结构+时间戳+迭代检索——比简单的向量记忆库更适合。另外，"拒答能力"（知道自己不知道）也是产品体验的关键：一个胡编乱造的 Agent 比一个说"我不记得"的 Agent 更伤信任。
