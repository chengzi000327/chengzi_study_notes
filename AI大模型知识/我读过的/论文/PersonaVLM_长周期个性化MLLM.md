## 一、认知进化：从"外挂提示词"到"神经网络硬编码"

传统 ReAct 架构依赖外部长 Prompt 约束，在 7B 中型模型上存在致命缺陷：长上下文迷失、格式崩溃、幻觉调用。

PersonaVLM 的核心突破：利用 **后训练强化学习（Post-training RL）**，将整个 ReAct 工具检索流直接固化/焊死为模型的参数条件反射。

### 神经层面的格式与逻辑规训（三个铁律奖励函数）

- **格式惩罚（Format Gate）：** 严格封锁 `<think>`、`<retrieve>`、`<answer>` 闭合标签。不按标准输出在梯度反向传播时直接被一票否决（零奖励）。
- **逻辑锚定（Consistency Constraint）：** 思考链（CoT）内容必须在数学上成为检索动作的"语义前置条件"。想查什么，必须先推理为什么查。
- **事实倒逼（Accuracy Incentive）：** 构造非检索不可得的硬核常识事实 QA，逼迫模型在参数空间内建立"模糊意图 → 本能触发检索"的依赖路径。

## 二、核心记忆（Core Memory）——稳定人设的定海神针

核心记忆与普通的向量检索记忆有着本质的不同，它的更新与检索机制才是整个智能体维持"稳定人设"的定海神针。

### 2.1 底层存储格式：明文 Key-Value JSON

核心记忆不使用复杂的密集向量（Embedding）存储，而是直接使用**高度结构化的明文键值对**，被严格划分为两大硬编码区块：

```json
{
  "HUMAN_PROFILE": {
    "name": "张三",
    "age": 28,
    "gender": "男",
    "core_preference": "深度科技发烧友，对花生过敏",
    "current_status": "刚从北京搬家到上海"
  },
  "SYSTEM_PERSONA": {
    "role": "专属 AI 助手（十七）",
    "tone_style": "理性、带有微幽默，面对用户焦虑时表现出极高情绪价值",
    "relationship_stage": "已相处半年，类似默契的极客搭档"
  }
}
```

### 2.2 "无感检索"机制：零成本常驻

**它根本不需要检索。** 普通的语义或情景记忆需要模型通过 `<retrieve>` 标签去数据库里捞，而核心记忆是**每一轮对话开始前，被外部框架硬编码、无条件直接塞进 Prompt 最顶部的 System 区域的**。

**为什么这么做？** 用户的名字、最核心的忌口（如花生过敏）、AI 自己的名字和语气，是绝对不能在长对话中因为上下文被刷掉或者漏检的。如果靠向量检索，一旦漏检，AI 就会问出"请问你叫什么名字？"这种直接让人出戏的致命降智问题。因此它作为全局最高优先级变量，每轮无条件加载。

### 2.3 "复盘式"异步更新机制（Overwrite）

核心记忆的更新是这篇论文的核心工程创新之一，采用"会话结束后的异步总结与覆盖写"策略：

```
用户说："我刚拿到上海一家大厂的 Offer"
        │
        ▼（正常对话回复，生成 <answer>）
        │
当前 Session 结束
        │
        ▼（触发后台异步复盘任务）
        │
智能体框架调用专门的"核心记忆提取参数"
让模型对比【当前对话】与【原有 Core 块】
        │
        ▼
发现冲突：旧 Core 写着"北京"，对话说"去上海"
        │
        ▼
触发覆盖写（Overwrite）：
  • 删除旧字段 `current_status: 北京`
  • 写入新字段 `current_status: 搬家到上海`
```

#### 更新机制的底层逻辑

1. **彻底消除身份冲突：** 核心记忆代表"不可分割的绝对事实"。如果采用只增不删的追加策略，下一次 System Prompt 就会变成 `[现居北京, 现居上海]`，模型就会精神分裂。必须通过物理覆盖（Overwrite），确保变量的绝对唯一性。
2. **高价值信息蒸馏：** 模型在异步复盘时，会过滤掉所有废话（如"今天天不错"），只把能改变 `HUMAN` 和 `PERSONA` 根本属性的金砂（如结婚、换工作、改偏好）提炼并改写进核心记忆块。

## 三、存储哲学：心理学架构与计算机工程的数据结构映射

长周期动态个性化的最大挑战：记忆臃肿（Context Explosion）与冲突（Information Contraction）。

论文将认知心理学对人类记忆的分类映射到计算机科学的数据结构策略（Mutable vs. Immutable）：

### 四类记忆的数据结构设计

**1. 核心记忆（Core Memory）**
- 心理定位：自我与客体认知
- 数据结构：结构化固定 Key-Value 字典
- 读写策略：覆盖写（Mutable），消灭冲突
- 加载机制：全局常驻变量（无需检索）
- 触发条件：用户修改或透露了最根本的社会/人设属性

**2. 程序记忆（Procedural Memory）**
- 心理定位：潜意识反射与预期
- 数据结构：状态机标识（Identifier）+ 第三人称短句
- 读写策略：智能归纳合并与更新（Mutable）
- 核心作用：行为预测与周期的主动配合

**3. 语义记忆（Semantic Memory）**
- 心理定位：百科知识谱系
- 数据结构：密集向量 + 明文 + 时间戳 + 视觉切块（Image Crop）
- 读写策略：纯追加（Additive）+ 时间衰减检索加权
- 核心作用：跨时间的偏好锚定（如换口味的软逻辑解决）

**4. 情景记忆（Episodic Memory）**
- 心理定位：时光编年史
- 数据结构：双层解耦（索引摘要层 + 原始日志层）
- 读写策略：历史封存，绝对只读不可篡改（Immutable）
- 核心作用：人际关系动力学变迁分析

### 冲突消灭与消化的双轨制

1. **硬物理删除（源头断绝）：** Core 和 Procedural 发生变动直接修改最新 Canonical Version，从工程上消灭冲突。
2. **软逻辑消化（大脑解耦）：** Semantic 和 Episodic 只增不删。
   - 时间衰减加权：检索时 $[\text{Similarity} \times \text{Time Decay Factor}]$，新事实排位靠前。
   - LLM 软化矛盾：模型同时查出旧偏好和新偏好时，由 RL 训练出的 `<think>` 大脑进行时序逻辑判断，主动吐故纳新。

### 四轨记忆联动闭环（完整示例）

> **用户多模态输入：** "（发了一张外卖生煎包的照片）今天搬家累瘫了，点个这个吃。"

1. **核心记忆（Core）直接加载：** 框架直接把 `[用户：张三，刚搬家到上海]` 和 `[AI：语气幽默的搭档]` 喂给模型。模型不需要检索就已经知道用户为什么在上海吃生煎包了。
2. **模型触发 ReAct 思考与检索（Semantic & Episodic）：**
   - 模型在 `<think>` 里想：他为什么说累瘫了？上个星期他提到过搬家细节吗？
   - 模型在 `<retrieve>` 里生成参数：查找上周关于 `搬家, 找房子` 的情景记忆（Episodic）。
   - 框架捞出上周日记："用户上周疯狂吐槽上海找房子租金太贵，打包行李很痛苦。"
3. **程序记忆（Procedural）预判：** 模型识别到"今天搬家"暗示了一个物理劳累事件，自动预判用户接下来可能需要休息推荐或外卖推荐。
4. **最终对齐输出（`<answer>`）：** 结合核心记忆的人设语气和情景记忆的前情提要，输出极高情商的回答：
   > "恭喜张三总算乔迁新居！上周听你吐槽打包行李像渡劫，现在可算能安顿下来吃顿正宗的上海生煎了。今晚好好睡一觉！"

**这篇论文最牛的地方：** 核心记忆负责常驻大局，程序记忆负责习惯预判，语义记忆负责事实卡片，情景记忆负责故事细节。四者各司其职，才把长周期的拟人个性化真正做了落地。

## 四、算法演进：算力内卷下的 RL 核心策略博弈

### PPO（绝对值导师制 · 旗舰标配）
- 现状：GPT-4o, Claude 3.7, Gemini Pro 等闭源巨头的底牌
- 本质：算力管够时，绝对价值估算在开放式生成中最稳健
- 痛点：需要 Critic 模型绝对打分，显存极重（常驻 4 个大模型）

### GRPO（组内相对排名 · 推理利器）
- 现状：DeepSeek-R1 核心基石，PersonaVLM 用来实现 7B 本地模型标签约束
- 本质：拿掉 Critic 模型，显存内零成本"组内相对论"
- 缺点：全组摆烂（全错）时失效

### GSPO（尖子生淘汰赛 · 逻辑硬核）
- 现状：大厂攻克顶级数学、高阶代码、黑盒环境规划时的前沿变体
- 本质：解决长思维链探索中"大量错误样本拉低全组平均分"的痛点
- 核心：稀疏过滤器将平庸回答权重归零，强制模型只向 Top-k 学习

### DPO（静态偏好映射 · 开源主流）
- 现状：Llama 3 系列微调、低成本平替
- 本质：将 RL 损失等价压缩到离线 SFT 阶段
- 代价：模型永远失去"自主探索（Exploration）"和"自我觉醒思维链"的上限

## 结语

用轻量级硬件实现长效、高情商、懂纪律的个人专属伴侣大模型的完整闭环方案。

PersonaVLM 证明了：大模型的未来不单单是参数量的暴力堆叠，通过格式化的 GRPO 算法重塑大脑，配合心理学级的四轨数据结构约束，一个 7B 的小模型同样能开辟出让通用巨头感到战栗的"高同理心"本地化智能新世界。
